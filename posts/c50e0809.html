容器诞生

物理宿主机部署集群特点

  • 部署慢

  • 成本高

  • 资源浪费

  • 难以迁移和扩展

  • 可能会被限定硬件厂商

虚拟化技术

  1. 一台宿主机
  2. 一个操作系统(win mac linux)
  3. cpu支持与开启 Hypervisor:支持虚拟化技术
  4. 通过 vmware Virtual KVM Docker 等工具创建多个虚拟化应用

虚拟化技术特点

  • 每一个虚拟都是完整的操作系统,耗资源高,要分配系统配置
  • 一台机器能创建多个虚拟机,资源利用率高
  • 等价: 程序在虚拟运行的表现一致
  • 安全: 虚拟机彼此隔离
  • 部署迁移克隆方便

综上所述,为每个独立环境给一个完整的系统仍然不是我们想要的,我们需要的是轻量级隔离环境,所以容器应运而生.

容器技术

  • 低耗: 容器内的应用程序直接运行在宿主机的内核上, 没有直接的内核.
  • 高效: 相比虚拟机需要的配置资源少很多
  • 高速: 相比虚拟机启动一个系统, 容器是秒级启动
  • 一致: 迁移不需要额外配置环境依赖
  • KVM需要硬件支持CPU的虚拟化支持, 而容器不需要Hypervisor

简化项目流程

Docker引擎

Docker Engine 是 C/S架构

Docker Daemon

Daemon是必须进程, 用于管理Docker, 包含:

  • 镜像 image

  • 容器 containers

    • 容器镜像是轻量的、可执行的独立软件包 ，包含软件运行所需的所有内容：代码、运行时环境、系统工具、系统库和设置。

    • 容器化软件适用于基于Linux和Windows的应用，在任何环境中都能够始终如一地运行。

    • 容器赋予了软件独立性，使其免受外在环境差异（例如，开发和预演环境的差异）的影响，从而有助于减少团队间在相同基础设施上运行不同软件时的冲突。

    • 容器就是将软件打包成标准化单元，以用于开发、交付和部署。

    • 容器和虚拟化的区别：

      通俗地讲，容器就是一个进程，它无法直接使用硬件资源，但自身又带着一堆类似于os的环境。从某种意义上讲，我们可以认为它是轻量级的虚拟化（当然，它和虚拟化最大的不同就在于无法直接使用硬件资源）。

  • 网络 network

  • 数据卷 Data Volumes

REST API

提供除命令行外的交互API接口

Docker Client

客户端使用RESTAPI和Daemon进行访问

Docker 平台组成

  • image: Docker 镜像是一个特殊的文件系统，除了提供容器运行时所需的程序、库、资源、配置等文件外，还包含了一些为运行时准备的一些配置参数（如匿名卷、环境变量、用户等）。镜像不包含任何动态数据，其内容在构建之后也不会被改变。
  • Container: 容器(里面装着目标应用程序)
  • dockerhub: 官方镜像仓库(类似于GitHub仓库)
  • dockerfile: 将部署项目的操作, 写成一个脚本.

镜像和容器 的关系就像是面向对象设计中 类和实例

Docker生命周期

Dockerfile

Dockerfile 是一个用来构建镜像的文本文件，文本内容包含了一条条构建镜像所需的指令和说明。

Docker 可以通过读取 Dockerfile 中的指令自动构建镜像。

  • FROM：指定基础镜像
  • RUN：执行命令
  • COPY：复制文件
  • ADD：更高级的复制文件
  • CMD：容器启动命令
  • ENV：设置环境变量
  • EXPOSE：暴露端口

变量

变量用 $variable_name 或者 ${variable_name} 表示

  • ${variable:-word}表示如果variable设置，则结果将是该值。如果variable未设置，word则将是结果。
  • ${variable:+word}表示如果variable设置则为word结果，否则为空字符串。

变量前加 *\* 可以转义成普通字符串：\$fooor\${foo}，表示转换为$foo和${foo}文字。

常用命令

服务

  • 版本详细信息: doucker version
  • 版本简要信息: docker -v
  • docker服务:
    • 开启|关闭|开机自启systemctl start|stop|enable docker
    • 重启|停止service docker restart|stop

镜像

  • 检索查找镜像: docker search bili
  • 拉取镜像: docker pull [选项] [Docker Registry 地址[:端口号]/]仓库名[:标签]
    • 如拉取一个b站自动签到项目 docker pull zai7lou/bilibili_tool_pro , 斜杠前面是地址即用户,后面是仓库名,仓库名没有标签会默认拉取最新的即 bilibili_tool_pro:latest
  • 列出本地镜像: docker image ls
  • 删除本地镜像: docker rmi <镜像Id>
  • 导出镜像: docker save
  • 导入镜像: docker load

容器

  • 查看运行中的容器: docker ps
  • 查看本机所有容器: docker ps -a
  • 新建并启动: docker run [镜像名/镜像ID]
  • 启动: docker start [容器ID]
  • 停止: docker stop [容器ID]
  • 终止: docker kill [容器ID]
  • 重启: docker restart [容器ID]
  • 删除: docker rm [容器ID]
  • 导出容器|容器快照: docker export [容器ID]
  • 导入容器: docker import [容器ID]
  • 容器日志: docker logs <可选参数> [容器ID]
    • -f : 跟踪日志输出
    • --since :显示某个开始时间的所有日志
    • -t : 显示时间戳
    • --tail :仅列出最新N条容器日志
