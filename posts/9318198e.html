Hexo一键多端部署

发表于2022-11-11|更新于2022-11-11|闲不住
|阅读量:

前言

本文目的

  • 最优雅的方式一行命令更新 Gitee+Github+Vercle

  • 未来会把语雀也加进来,做到增量网络与本地互相同步,未增量则比较时间进行覆盖

三个默认

  • 本文默认你会基本使用 Hexo 并能成功运行(哪怕是demo)
    不会的点击学习 -> Hexo官方 -> markdown语法大全
  • 本文默认你已经Gitee实名认证或有自己域名
    没有的去认证 -> 帐号管理 - Gitee.com
  • 本文默认你能解决某种科学网络问题
    不能的话我也不能教你
    你就部署gitee+随缘github吧

Gitee Page

  1. 打开gitee 我的工作台 - Gitee.com

  2. 创建一个仓库

  3. 修改仓库路径

  4. 生成ssh公钥

    1. 配置ssh账户与邮箱, 邮箱随意哪个, 不必与git账户系相同

    2. 生成ssh

  5. 添加ssh公钥

    1. 查找生成的公钥

    2. 添加到 gitee 或 github, 一般都在 账户设置->ssh->添加ssh公钥 之类的里面, 公钥标题顺便取(当然最好明确知道是干嘛的)

    3. 测试是否添加成功

  6. 在hexo运行目录的 _config.yml 最后加上git配置

  7. 安装同步包

  8. 奉上 hexo一键三连

  9. 开启Page服务 (每次更新完博客记得来这里点一下更新)

最后生成如下东西, 也就是hexo的public目录下内容

Github Pages

  1. 建公共仓库,且名字为 你的github名字.github.io (是告诉GitHub我不要后缀,不然就会成为 https://thatcoders.github.io/巴拉巴拉一堆/ 而不是 https://thatcoders.github.io/ . 简洁一点不好吗??? 是吧)

  2. 添加ssh公钥

  3. 在hexo运行目录的 _config.yml 最后加上git配置

    这里是github+gitee双线部署, 不要gitee可以自行删去

  4. 安装同步包(gitee时候已安装可跳过)

  5. 奉上 hexo一键三连(如果两个git都配置了,会一起推送)

GitHub默认为 gh-pages 分支开启 pages服务, 更新也不用去手动点更新, 以上GitHub结束

Vercel Serve

  1. 新建项目
  2. 选择GitHub导入 用户名.github.io 的那个项目
  3. 不用配置直接部署即可
  4. 项目的设置->域名(domin)->绑定自己的域名并去服务商解析即可(对了这个方法可以忽略域名备案嘿嘿)
